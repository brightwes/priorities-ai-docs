# Tasks

**Route:** `/prioritization/my-tasks`

The Tasks page is your personal work queue inside Priorities.ai. It aggregates everything assigned to you or created by you across the workspace so you have one place to track open work without needing to dig through individual items or cycles.

## What you see

The page shows tasks in three views you can switch between:

| View | What it shows |
|---|---|
| **Assigned to me** | Tasks where you are the assignee |
| **Created by me** | Tasks you created, regardless of who owns them |
| **All my tasks** | Combined view of both |

Each task row shows the task name, the item or context it is attached to, its due date, and its current status.

## Task states

Tasks move through a simple status flow:
- **Open** — not yet started
- **In progress** — actively being worked
- **Done** — completed

## Actions

| Action | How |
|---|---|
| Create a new task | Click **New Task** in the top right |
| Open a task detail | Click any task row |
| Mark a task complete | Toggle the checkbox or open the task and change its status |
| Assign a task to someone | Open the task and set the assignee field |
| Link a task to an item | Open the task; the item link is shown in the context section |

## Task Runner

Clicking a task opens the **Task Runner** — a focused view for working through a task step by step. The runner shows the full task description, any instructions attached, a checklist if one was defined, and the ability to record a completion note.

## Tips

- Tasks can be created from any item's detail page as well as from this page.
- The sidebar badge on the Tasks nav link shows the count of open tasks assigned to you — a quick way to see if anything is waiting.
- Tasks are workspace-scoped; they are not tied to a specific cycle unless you link them to a cycle-owned item.
