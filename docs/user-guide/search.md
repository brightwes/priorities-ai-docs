# Search

**Route:** `/prioritization/search`

The Search page is a full-featured advanced search that lets you find anything in the workspace — items, cycles, sessions, strategies, and more — using filters, free text, and saved searches.

## What you see

The page has two main areas:
- **Filter panel** (left) — a set of filter facets you can combine
- **Results area** (right) — matching records shown as cards or rows

## Filter facets

You can filter by any combination of:
- **Type** — item, cycle, session, strategy, objective, etc.
- **Status** — active, archived, complete, draft, etc.
- **Owner / assignee** — person who owns or is assigned to the record
- **Labels / tags** — any labels applied to items
- **Category / item class** — item type classification
- **Date ranges** — created, modified, due dates
- **Custom attributes** — any workspace-defined attribute fields

Multiple filters combine with AND logic. Within a single facet, multiple values combine with OR logic.

## Actions

| Action | How |
|---|---|
| Run a search | Type in the search box or apply filter facets |
| Save a search | After applying filters, click **Save search** and give it a name |
| Load a saved search | Select from the saved searches list that appears when the search box is focused |
| Open a result | Click any result card to navigate directly to that record |
| Clear all filters | Click **Clear** above the filter panel |

## Saved searches

Saved searches let you bookmark a filtered view and return to it later. Once saved, a search appears in the saved searches list accessible from the search bar. Saved searches are personal — they are not shared with the workspace by default.

## Tips

- The **Command palette** (`⌘ K`) gives you a faster way to jump to specific records by name without going to the Search page.
- You can reach the Search page from the sidebar at any time; filters you applied persist during your session.
- Saved searches also appear as **pinned views** in the sidebar (for item list searches that you star).
